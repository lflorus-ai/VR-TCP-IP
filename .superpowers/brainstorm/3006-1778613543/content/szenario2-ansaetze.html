Spieler: geht zum Computer → liest welches Paket fehlt → geht zurück zum Regal → holt das spezifische Paket → legt es auf die Palette

A — Computer-Fehlermeldung + physische Neulieferung

Am Computer erscheint eine Fehlermeldung (wie ein TCP ACK-Timeout). Spieler muss das verlorene Paket aus dem Regal holen und korrekt neu einlegen. Stärker physisch, klare Analogie zum Lagerhaus-Thema.

Stärken

  • Passt zur Lagerhallen-Metapher
  • Klares Interaktionsmuster (identisch zu S1)
  • Computer als Empfänger-Interface erkennbar

Schwächen

  • Weniger Abwechslung gegenüber S1
  • Paketverlust fühlt sich nicht "zufällig" an

HUD zeigt sofort welche Pakete "verloren" gingen. Spieler weiß ohne Computer-Besuch was zu tun ist.

B — HUD-gesteuerter Neuübertragungsauftrag

Nach S1 blinkt das HUD: Paketverlust erkannt, 2 Pakete müssen neu gesendet werden. Die fehlenden Pakete erscheinen wieder im Regal (oder in einem separaten Regal B). Spieler muss sie erneut sortieren.

Stärken

  • Flüssigerer Übergang von S1
  • Kein Szenenwechsel nötig
  • Sofort klar was zu tun ist

Schwächen

  • Computer-Element im Büro wird ignoriert
  • Weniger dramatischer "Moment"
