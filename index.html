Das Grundprinzip: Aus den ersten Oktetten einer IP-Adresse lässt sich das Zielnetz ableiten — genau wie eine Postleitzahl die Region bestimmt.
🌐 Was du getan hast — Schicht 3 (IP)

Du hast wie ein Router gearbeitet. Das IP-Protokoll (Schicht 3) macht genau das:

  • Jedes Datenpaket trägt eine Ziel-IP-Adresse
  • Router lesen den Netzwerkpräfix (erste Oktette) und leiten weiter
  • IP arbeitet verbindungslos — jedes Paket findet seinen eigenen Weg
🔵 10.x.x.x — Klasse A privates Netz 🟢 172.16.x.x — Klasse B privates Netz 🟠 192.168.x.x — Klasse C privates Netz

🔁 Schicht 4 · TCP — Zuverlässigkeit

IP (Schicht 3) liefert Pakete nach bestem Bemühen, aber ohne Garantie. TCP (Schicht 4) fügt Zuverlässigkeit hinzu:

  • Bestätigung (ACK): Empfänger bestätigt jedes Paket
  • Retransmission: Fehlende Pakete werden erneut gesendet
  • Sequenznummern: TCP stellt die richtige Reihenfolge sicher

Deine Aufgabe: verlorene Pakete aus dem Regal holen und erneut zustellen.

Lern-Szenario 2 · Abgeschlossen
📡
Neuübertragung erfolgreich!
Gesamtpunkte— Neugesendet—
Genau so funktioniert TCP: Fehlende Pakete werden vom Empfänger erkannt und vom Sender erneut übertragen — das nennt man Retransmission. Ohne diesen Mechanismus wäre verlässliche Kommunikation im Internet nicht möglich.
🔁 Was du getan hast — Schicht 4 (TCP)
  • Empfänger erkennt fehlende Pakete → sendet kein ACK (Bestätigung)
  • Sender überträgt fehlende Pakete erneut (Retransmit)
  • TCP nutzt Sequenznummern, um Reihenfolge beim Empfänger sicherzustellen
  • Erst wenn alle Pakete bestätigt sind, gilt die Übertragung als vollständig
TCP ✅ Zuverlässig ✅ Reihenfolge garantiert ⚠️ Mehr Overhead z.B. HTTP, E-Mail
UDP ⚡ Sehr schnell ❌ Keine Garantie ✅ Wenig Overhead z.B. Video-Streaming, DNS
