Szenario 2 · Transport Layer
📠
Eingehende Meldung!
Der Empfänger meldet: Einige Pakete sind nicht angekommen.
🔁 Schicht 4 · TCP — Zuverlässigkeit

IP (Schicht 3) liefert Pakete nach bestem Bemühen, aber ohne Garantie. TCP (Schicht 4) fügt Zuverlässigkeit hinzu:

  • Bestätigung (ACK): Empfänger bestätigt jedes Paket
  • Retransmission: Fehlende Pakete werden erneut gesendet
  • Sequenznummern: TCP stellt die richtige Reihenfolge sicher

Deine Aufgabe: verlorene Pakete aus dem Regal holen und erneut zustellen.

Lern-Szenario 2 · Abgeschlossen
📡
Neuübertragung erfolgreich!
Gesamtpunkte— Neugesendet—
Genau so funktioniert TCP: Fehlende Pakete werden vom Empfänger erkannt und vom Sender erneut übertragen — das nennt man Retransmission. Ohne diesen Mechanismus wäre verlässliche Kommunikation im Internet nicht möglich.
🔁 Was du getan hast — Schicht 4 (TCP)
  • Empfänger erkennt fehlende Pakete → sendet kein ACK (Bestätigung)
  • Sender überträgt fehlende Pakete erneut (Retransmit)
  • TCP nutzt Sequenznummern, um Reihenfolge beim Empfänger sicherzustellen
  • Erst wenn alle Pakete bestätigt sind, gilt die Übertragung als vollständig
TCP ✅ Zuverlässig ✅ Reihenfolge garantiert ⚠️ Mehr Overhead z.B. HTTP, E-Mail
UDP ⚡ Sehr schnell ❌ Keine Garantie ✅ Wenig Overhead z.B. Video-Streaming, DNS

Lern-Szenario 5 · Internet-Schicht
🌐
IP-Routing: Pakete auf dem richtigen Weg
Jetzt geht es um die Internet-Schicht (IP): Jedes Paket trägt eine Ziel-IP-Adresse — der Router liest diese Adresse und entscheidet, welchem Netz das Paket gehört.
🌐 Schicht 2 · IP — Adressierung & Routing
  • IP-Adresse: z.B. 192.168.1.10 — eindeutige Adresse im Netz
  • Subnetz: Die ersten 3 Oktette bestimmen das Netz (z.B. 192.168.1)
  • Routing: Der Router ordnet jedes Paket dem richtigen Zielnetz zu

Deine Aufgabe: Geh zurück in die Lagerhalle — nimm ein Paket aus dem Regal ([E]), prüfe die IP-Adresse im Lieferschein und lege es auf die richtige Palette ([E]).

Szenario 5 · Fehlerbehandlung
📡
Ein Paket ist verloren gegangen!
Du hast alle Pakete korrekt geroutet — trotzdem kann im Netz ein Paket verloren gehen (z.B. durch Überlastung oder Störungen auf dem Weg).
📡 Was jetzt passiert — TCP-Neuübertragung
  • Der Empfänger bemerkt das fehlende Paket (keine Bestätigung / ACK)
  • Das Paket wird deshalb erneut angefordert
  • Deine Aufgabe: Hol das verlorene Paket und liefere es noch einmal aus

Genau das macht TCP automatisch — verlorene Pakete werden so lange neu gesendet, bis sie ankommen.

Lern-Szenario 5 · Abgeschlossen
🌐
Alle Pakete korrekt zugestellt!
Punkte—
Genau so funktioniert IP-Routing: Jedes Datenpaket wird anhand seiner Ziel-IP-Adresse dem richtigen Netz zugeordnet — genauso wie du gerade Pakete nach Subnetz sortiert hast.
🌐 Was du getan hast — Internet-Schicht (IP)
  • IP-Adresse lesen und das Subnetz bestimmen
  • Paket dem richtigen Zielnetz zuordnen (= Routing)
  • Router trifft diese Entscheidung für jedes Paket automatisch

S3 — Transportschicht
TCP oder UDP?
Die Transportschicht regelt die End-zu-End-Übertragung. Zwei Protokolle stehen zur Wahl — je nachdem, ob Zuverlässigkeit oder Geschwindigkeit wichtiger ist.
TCP
Zuverlässig & bestätigt
Verbindungsorientiert · garantiert Zustellung · jedes Paket wird bestätigt — für Dateien, E-Mails, Webseiten.
UDP
Schnell & verbindungslos
Keine Bestätigung · kurze Verluste okay · sehr schnell — für Video-Streams, Gaming, VoIP, DNS.
